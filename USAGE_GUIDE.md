# 运输车队实时追踪系统 - 使用指南

## 🎯 功能特性

### 核心功能
- ✅ **实时位置追踪**: 车辆位置每15秒更新一次
- ✅ **轨迹显示**: 显示车辆运动轨迹（虚线）
- ✅ **多车辆支持**: 同时追踪多个车辆
- ✅ **车队管理**: 查看所有车辆的实时状态
- ✅ **搜索功能**: 快速搜索特定车辆
- ✅ **电池监控**: 实时显示车辆电池状态
- ✅ **信号强度**: 显示MQTT信号强度

### 模拟硬件
- 🚚 **3个模拟车辆**: vehicle_001, vehicle_002, vehicle_003
- 📍 **合理路线**: 北京 → 天津（约120km）
- ⏱️ **15秒上报**: 每15秒上报一次位置
- 🚗 **80 km/h速度**: 模拟真实运输车速度
- 🔋 **电池模拟**: 模拟电池消耗过程
- 📶 **信号模拟**: 模拟信号强度变化

## 🚀 快速启动

### 方式1: 一键启动（推荐）

```bash
python run_all.py
```

这将同时启动：
- 后端服务 (FastAPI)
- 前端应用 (Vue 3)
- 模拟硬件 (3个车辆)

### 方式2: 分别启动

**终端1 - 启动后端:**
```bash
cd backend
python main.py
```

**终端2 - 启动前端:**
```bash
cd frontend
npm run dev
```

**终端3 - 启动模拟硬件:**
```bash
cd backend
python simulate_vehicle.py multi 3
```

## 🌐 访问应用

启动后，在浏览器中打开：

## 🎮 交互操作

### 地图操作
1. **点击车辆标记**: 显示车辆详细信息弹窗
2. **点击车辆列表**: 选中车辆并高亮显示
3. **拖动地图**: 平移地图视图
4. **滚轮缩放**: 放大/缩小地图

### 轨迹管理
1. **显示轨迹**: 点击"🛣️ 显示轨迹"按钮
2. **隐藏轨迹**: 再次点击按钮隐藏
3. **轨迹颜色**: 每个车辆有不同颜色的轨迹

### 搜索功能
1. 在左侧搜索框输入车辆ID
2. 列表自动过滤显示匹配的车辆
3. 清空搜索框显示所有车辆

## 📈 数据说明

### MQTT消息格式
```json
{
  "device_id": "vehicle_001",
  "latitude": 39.9042,
  "longitude": 116.4074,
  "battery": 85,
  "signal_strength": -65,
  "status": "active",
  "timestamp": "2026-03-12T10:30:00Z"
}
```

### 模拟路线
```
北京市中心 (39.9042, 116.4074)
    ↓
北京郊区 (40.0000, 116.6500)
    ↓
天津市中心 (39.0842, 117.2010)
```

### 速度计算
- 速度: 80 km/h
- 上报间隔: 15秒
- 每次移动距离: 80 * 1000 / 3600 * 15 ≈ 333米

## 🔧 自定义配置

### 修改车辆数量
```bash
python simulate_vehicle.py multi 5  # 启动5个车辆
```

### 修改上报间隔
编辑 `backend/simulate_vehicle.py`:
```python
self.interval = 15  # 改为其他值（秒）
```

### 修改速度
编辑 `backend/simulate_vehicle.py`:
```python
self.speed_kmh = 80  # 改为其他值（km/h）
```

### 修改路线
编辑 `backend/simulate_vehicle.py` 中的 `generate_route()` 方法

## 📱 API端点

### REST API

| 方法 | 端点 | 说明 |

| GET | /health | 健康检查 |

## 🐛 故障排除

### 地图不显示
- 检查浏览器控制台错误
- 确保WebSocket连接成功
- 检查后端是否运行

### 车辆不更新
- 检查模拟硬件是否运行
- 查看后端日志
- 检查MQTT连接状态

### 前端无法连接
- 检查后端是否运行在8000端口
- 检查防火墙设置
- 查看浏览器控制台错误
